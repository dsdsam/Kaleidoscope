The Rule Driven Input Generator

The Rule-Driven Input Simulating Program (RDISP) is, in general, a DsdsseCell input generator, representing the case when the DsdsseCell input depends on its current state.

In many modeled cases, the state of the DsdsseCell (or, the value the DsdsseCell takes) is strictly defined as the function of state of some other Cells and can be changed only when the state of the other Cells is changed. Such state is considered to be a stable one. Meanwhile, sometimes there may exist some unstable states. These states can be changed spontaneously, without any visible reasons.

The DsdsseCell's spontaneous transition from some particular state to another particular state is simulated with RDISP. This program is specified as a set of rules, and each rule is described as triplet:

Conditional State-Ticks-Target State.

//

Ticks-State.

The Conditional State is the unstable state at which the spontaneous transition takes place. The Ticks is the number of simulating ticks, which have to pass before the transition happens, so, they represent the life time of the unstable state. And the Target State is the state, which has to replace the unstable one.

To create the Rule-Driven Input Simulating Program you have to:

  1. insert into the table desired number of rules (rows);

  2. select one of the rules to define it;

  3. pick up the state value from the Allowed States List to set the state value for the rule's Conditional State;

  4. set the amount of ticks for the selected rule according to the format described below;

  5. pick up the state value from the Allowed States List to set the state value for the rule's Target State;

  6. repeat items 2 through 5 for all program rules.
