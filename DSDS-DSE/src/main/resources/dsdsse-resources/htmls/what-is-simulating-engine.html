The Simulating Engine

The Simulation Engine is the DSE component that can simulate developed model behavior over time and represent the simulation steps (ticks) on the screen. It simulates the model behavior by cyclically analyzing the model current state, recognizing current situations and mapping the situations to the dependent on the situations states, that the properties should take at the next to the current moment of time.

To enable Simulation Engine user sets DSE to Simulation mode by selecting menu item Simulation -> Set Simulation Mode.

The Simulation Engine has four buttons located on the Design Space Toolbar: Start, Stop, Step, and Reset. The Start, Step, and Reset buttons default state is enabled, while state of Stop button is disabled.

Clicking on Start button starts auto simulation process. This click disables Start and Step buttons while enabling Stop button.

Clicking on Stop button stops auto simulation process and returns Start and Step buttons back to their enabled state.

Clicking on Step button executes only one simulation step at a time.

Clicking on Reset button resets model to its initial state.

Located below Design Space simulation analysis tools allow user to select all or subset of model Property Nodes and analyse model behavior over time. The Model Current State Panel presents each selected Property Node current state and provides its interpretation in the form of statement that includes property name as the statement's subject and describes the value of the current state as the statement's predicate. The Simulation Trace Log Panel presents selected states history over simulation time. Both panels are updates after each simulation tick in real time.


The Presentation Shows & Help menu item:
"How to Run Model Simulation"
provides short and self-driven presentation on the process of model simulation.
