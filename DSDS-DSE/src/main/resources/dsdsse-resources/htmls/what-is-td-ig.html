The Time Driven Input Generator

The Time-Driven Input Simulating Program (TDISP) is, in general, a DsdsseCell input generator, providing the sequence of states the DsdsseCell takes over time.

As far as, the simulating runs in discrete steps termed ticks, the program is described as the sequence of pairs:

Ticks-State.

The Ticks, here, is the number of simulating ticks which have to pass before the next input takes place, and the State is the value of the input expected.

You can specify that the first step of the sequence is the program's phase. The phase step is accomplished only ones when the simulation starts. All other steps are accomplished one after another, repeatedly.

To create the Time-Driven Input Simulating Program you have to:

  1. specify if the program has phase by checking the box Has Phase;

  2. insert into the table desired number of steps (rows);

  3. select one of the steps to define it;

  4. set the amount of ticks for the selected step according to the format described below;

  5. pick up the state value from the Allowed States List to set the state value for the step;

  6. repeat items 3 through 5 for all program steps.

---------- || ----------

Both, the Time-Driven and the Rule-Driven ISP have the same format for the Ticks column. It is either

XXX
or,
R = XXX

The first format is used to describe that the amount of ticks is defined as a constant and XXX is the value of the constant.

The second format is used to specify that the amount of ticks is defined as a random number and XXX is that number's range.

The reasonable interval for value XXX is established as:

0<XXX<=999.

Entering anything other then format allows is considered as an error, and causes the table cell is highlighted in red. The error character is placed in brackets, e.g. R/34[A], and, the error message dialog box springs up informing you that the wrong input was made. When the box is closed, the character is automatically deleted from the editor.
