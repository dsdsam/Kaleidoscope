Kaleidoscope Project

Kaleidoscope is a research and development project intended to explore and present Qualitative Discrete Symbolic Dynamical Systems modeling and simulating technique based on Multicolored Logical Net Modeling Formalism. Currently the project consists of three major parts:

  1. Multicolored Logical Net Modeling Formalism (McLN MF).
  2. Space Exploration Mission (SEM) desktop application.
  3. Discrete Symbolic Dynamical Systems Development & Simulating Environment (DSDS DSE) desktop application.

Web site Discrete Symbolic Dynamical Systems & Models is the project's site presenting Multicolored Logical Net Modeling Formalism. The site page "McLN Viewer" presents several Discrete Symbolic Dynamical System models developed with the Multicolored Logical Net modeling formalism. All models are working examples. User can select a model from the "Examples" menu. Located on the page control buttons: "Start", "Stop", and "Step" allow user to start, stop, or do step-by-step simulation. "Reset" button resets model and simulation process into initial state. Another site page "McLN MF" provides a short and informal introduction to Multicolored Logical Net Modeling Formalism, including the description of the McLN model structure and behavior.

This site provides the description of two Kaleidoscope Project applications:

Space Exploration Mission is a robot behavior simulator Java desktop application. The Application is an imaginary Mission Control and Monitoring Center (MCMC) interacting with the remote robot - Rover. The task of the robot is to explore the designated space, collect environmental parameters and transmit Rover's current coordinates packed with collected data to MCMC in accordance with scheduled session. The robot can be controlled either via manual controls or by McLN-based Robot Behavior Controller. The application screen layout is presented in Figure 1.


Figure 1. Space Exploration Mission application screen snapshot.

Space Exploration Mission application screen is the imitation of the Mission Control and Monitoring Center dashboard. It consisting of following control and monitoring panels: (1) Manual Robot Control panel (left panel), (2) McLN-based Robot Behavior Controller (center panel), (3) Robot Observation Monitors - the cluster of monitors occupying right part of the screen, displaying a) the remote world and accomplishing its exploratory mission robot 3D views, b) the robot's view - the world view as it is seeing from the cabin of the Rover, and (5) Running Text Panel - the panel located below the monitor panels presenting received from the Rover remote data.

Utilized by the Robot Behavior Controller McLN model is developed by the Discrete Symbolic Dynamical Systems Development & Simulating Environment application described below.

Discrete Symbolic Dynamical Systems Development & Simulating Environment is a full fledged Java desktop application that utilizes McLN modeling formalism to design, develop and simulate qualitative dynamical system models. Respectively, the application can be set to operate in either Development or Simulation mode. By default application is started in Development mode. Figure 2 represents DSDS DSE initial screen - when the application is just started.


Figure 2. Started DSDS Development & Simulating Environment.

The application screen layout includes following (from top to bottom): Menu a Toolbar, Application Status panel, Design Space, Assisting Tools Area, and One Line Message panel. Design Space is the McLN Graph View holder. In usual case McLN Graph View occupies all Design Space area. Optionally, in addition to McLN Graph View, user may open three right side panels, such as: Initialization Assistant, Help Topics, and Preferences Setup panels. Print Preview panel may optionally take left and central space on the screen presenting Print Control panel on the left and Print Preview panel in the center of the screen.

Development Mode

While in Development mode the Environment allows McLN model developer to create McLN Graph, set Properties initial states, initialize Arcs with their Expected or Generated states, save partially created or complete model on hard drive, retrieve it, modify, and save again. The application produces McLN file that is intended to be uploaded by Space Exploration Mission application into its Robot Behavior Controller where it is used to take Rover sensors input, recognize dynamically arising situations, make real-time control decisions, and communicate with robot's local controllers. Figure 3 represents Project Space filled with developed McLN model graph in the center of the screen and The Initialization Assistant - wizard like tool - open on the right side of the Design Space.


Figure 3. DSDS Development & Simulating Environment in Development mode.

User switches DSE to Development mode by selecting menu item Development -> Set Development Mode. Switching to Development mode enables Creation and Modification menus. Available under Creation menu operations are create: Property Nodes, Condition Nodes, Polyline Arcs, Spline Arcs, and Simple Fragment. Available under Modification menu operations are: Move Model Elements, Move Model Fragments, Move Entire Model, and Delete Model Elements.

Simulation Mode

While in Simulate mode the Environment allows McLN model developer to test developed model behavior. When switching to Simulation mode the Assisting Tools Area is filled with following panels: Property Selection panel, Property Current State panel, and Execution History Trace Log panel. These are currently available tools assisting in the McLN model development, debugging and verification. Figure 4 represents the DSDS DSE screen snapshot when the Environment is in Simulation mode.


Figure 4. DSDS Development & Simulating Environment in Simulation mode.

User sets DSE to Simulation mode by selecting menu item Simulation -> Set Simulation Mode. Enabled in the Simulation mode toolbar buttons: Start, Stop, Step, and Reset allow developer to control simulation process. Clicking on Start button starts auto simulation process. This click disables Start and Step buttons while enabls Stop button. Clicking on Stop button stops auto simulation process and returns Start and Step buttons back to their enabled state. Clicking on Step button executes only one simulation step at a time. Clicking on Reset button resets model to its initial state.

Located below Design Space simulation analysis tools allow user to select all or subset of model Property nodes and analyse model behavior over time. The Model Current State Panel presents each selected Property node current state and provides its interpretation in the form of statement that includes property name as the statement's subject and describes the value of the current state as the statement's predicate. The Execution History Trace Log panel presents selected states history over simulation time. Both panels are updates after each simulation tick in real time.

As it was mentioned above some auxiliary panels may be optionally open on the right side of the Design Space panel. Presented in Figure 4 Simulation mode screen snapshot is the case when the Help Topics panel is open from "Presentation Shows & Help" menu and filled with selected from the menu topic: Running Model Examples.

Printing Developed Model

DSDS DSE makes user to be able to print developed model. To activate printing facility user opens Project menu and selects Print Preview Panel item or alternatively clicks corresponding toolbar button. Figure 5 represents DSDS DSE screen when the Environment is set to print developed McLN model. For the purpose of illustration snapshot also presents Property Setup panel open on the right side of the screen.


Figure 5. DSDS Development & Simulating Environment is set to print developed McLN model.

As it is presented on the snapshot, the printing facility allows user to print in Portrait or Landscape format. The printed image is scaled to fit standard A4 page.
